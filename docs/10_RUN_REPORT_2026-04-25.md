## 2. المشاكل التي تم حلها في هذه الجلسة

### 2.1 QGC: "Unknown 193:0"
**السبب**: عدم إرسال `AUTOPILOT_VERSION` (msg 148) أو إرساله بترتيب حقول خاطئ.  
**الحل**:
- إضافة `send_autopilot_version()` في [run_sitl_qgc_udp.py](m13/6DOF_v4_pure/sitl/run_sitl_qgc_udp.py)
- ترتيب MAVLink الصحيح: حقول مرتبة **تنازلياً حسب الحجم** (uint64 → uint32 → uint16 → uint8[])
- `flight_sw_version = (1<<24) | (17<<16) | (0<<8) | 255` لـ "PX4 1.17.0 OFFICIAL"
- إرسال 3 مرات upfront + كل 5 ثوانٍ من thread الـheartbeat

### 2.2 QGC: "Sensor Status — All Errors"
**السبب**: في `SYS_STATUS` كان `sensors_health = 0x00000000` بينما `sensors_present = 0xFFFFFFFF`.  
**الحل**: ضبط `sensors_health = sensors_present = 0x3FFFFFFF` → كل الحساسات تظهر **خضراء**.

### 2.3 Range Bias +20.86m (SITL)
**السبب**: نموذج MPC يطلق نقطة الإصابة **+20.86m** بعد المدى المطلوب بسبب bias ديناميكي (drag/thrust).  
**الحل**: ضبط `target.drag_compensation_m: -15.8` في [6dof_config_advanced.yaml](m13/6DOF_v4_pure/config/6dof_config_advanced.yaml).  
**النتيجة**: 3020.86m → **3000.77m** (تحسّن **96%**).

### 2.4 SITL Slow (331s wall-clock)
**السبب**: `c_generated_code/Makefile` يعيد بناء `.so` كل تشغيل + MHE solve بطيء.  
**الحل**: cache hash موجود مسبقاً في [m130_mpc_autopilot.py](m13/6DOF_v4_pure/mpc/m130_mpc_autopilot.py) — يحفظ `.mpc_source_hash` ويتخطى البناء إذا لم يتغير المصدر.  
**النتيجة**: 331s → **103s** (أسرع **3×**).

### 2.5 تكرارية النتائج (Determinism)
**الحل**: `SITL_NOISE_SEED = HITL_NOISE_SEED = 20260424` — كل تشغيل يُنتج نفس النتيجة بالضبط.

## 4. خيارات التشغيل بالتفصيل

### 4.1 SITL (Linux native)
```bash
cd m13/6DOF_v4_pure
bash run_linux_sitl.sh
```
**ما يحدث**:
1. إنهاء أي QGC قديم + مسح cache
2. تصدير `ACADOS_SOURCE_DIR` و `LD_LIBRARY_PATH`
3. بناء MPC (`m130_rocket`) + MHE (`m130_mhe`) إذا تغير المصدر فقط
4. تشغيل QGC AppImage في الخلفية (PID مسجل)
5. انتظار 8s pre-flight (heartbeats + AUTOPILOT_VERSION + HOME_POSITION + SYS_STATUS)
6. تشغيل المحاكاة 6DOF بـ `dt=0.01s, MPC@50Hz, MHE@50Hz`
7. بث MAVLink عبر UDP 14550 (~10K رسالة لكل طيران)
8. حفظ CSV + HTML report
9. إبقاء QGC شغال 20s لرؤية النتيجة

**المتغيرات البيئية**:
- `SITL_DURATION` (افتراضي capped to 20s إذا duration > 30s)
- `SITL_NOISE_SEED` (افتراضي 20260424 للتكرارية)

### 4.2 HITL (Phone PX4)
```bash
cd m13/6DOF_v4_pure
HITL_KEEP_RUNS=99 bash hitl/run_phone_hitl.sh
```
**ما يحدث**:
1. كشف الجهاز عبر `adb devices`
2. تشغيل APK على الهاتف (`com.ardophone.px4v17`)
3. `adb forward tcp:5760 tcp:5760` (PX4 MAVLink)
4. تشغيل QGC على UDP 14550 (يصلها bridge من TCP)
5. `run_hitl_serial.py` يُحاكي المستشعرات → يبعث للهاتف عبر TCP
6. الهاتف يحسب MPC/MHE ويرد بأوامر servo
7. حفظ النتائج في `results/hitl_tcp/run_<timestamp>/`
8. **معيار الجودة**: `apogee >= 105m AND range >= 2900m AND retries <= 3`

**المتغيرات البيئية**:
- `HITL_NOISE_SEED=20260424` (مطابق SITL)
- `HITL_KEEP_RUNS=99` (يحفظ كل النتائج، 0=يحذف الفاشلة)
- `HITL_INJECT_NOISE=1` (افتراضي مفعّل)

### 4.3 محاكاة بدون QGC (سريعة للاختبار)
```bash
cd m13/6DOF_v4_pure
python3 rocket_6dof_sim.py
```

## 5. تحليل MHE (Moving Horizon Estimator)

### الإعدادات الحالية ([6dof_config_advanced.yaml](m13/6DOF_v4_pure/config/6dof_config_advanced.yaml))
- `mode: 'mhe'`
- `N_horizon: 20` (نافذة 0.4s)
- `dt: 0.02s` (50 Hz)
- `state_dim: 17` (pos + vel + quat + bias_accel + bias_gyro + ω̄)

### الأداء المُلاحظ
- **Solve time avg**: 19.4 ms (محسّن من 40.2 ms — ~2× أسرع)
- **Quality mean**: 0.32 (< 0.5 = ممتاز)
- **MHE تتقاطع مع MPC**: يعمل MHE قبل MPC في كل خطوة، يُحدّث الحالة المُقدّرة، MPC يستخدمها كـ initial guess.

### المستشعرات المُحاكاة
- IMU @ 400 Hz (accel σ=0.05, gyro σ=0.0025)
- GPS @ 20 Hz (pos σ=1.25m, vel σ=0.30 m/s)
- Baro @ 50 Hz (σ=0.60m)
- Biases ثابتة عبر التشغيل (seed-based)

## 8. القضايا المتبقية (Cosmetic، لا تؤثر على الطيران)

1. **`Vehicle did not respond to parameters`** — bridge لا يدعم MAVLink param protocol (msg 20/21/22). غير مطلوب للمحاكاة.
2. **`SQP_RTI: ACADOS_MINSTEP` بعد الإصطدام** — solver يحاول حل QP بعد الانتهاء؛ لا يؤثر على النتائج.
3. **QGC firmware warning ("supports v1.4.1+")** — كان بسبب ترتيب الحقول، تم حله الآن.
4. **`SITL report bundle failed: No module named 'sitl_report'`** — module مفقود لتجميع PDF؛ HTML report يعمل.
