# 1. ما هو ACADOS؟

## تعريف

ACADOS (**A**dvanced **CA**sADi-based tools for **D**ynamic **O**ptimization and **S**imulation) هي مكتبة مفتوحة المصدر لحل مسائل التحكم الأمثل (Optimal Control Problems) بسرعة عالية. طُوّرت في جامعة فرايبورغ بألمانيا.

**الهدف:** أعطني معادلات حركة نظامك + ماذا تريد تحقيقه + ما القيود، وأنا أعطيك أفضل أوامر تحكم.

## المكونات الأساسية

```
┌─────────────────────────────────────────────────────────────┐
│               ما تكتبه أنت بـ Python (مرة واحدة)            │
│                                                             │
│  ┌─────────────────────┐    ┌────────────────────────────┐  │
│  │ نموذج CasADi        │    │ إعداد OCP                  │  │
│  │ معادلات الصاروخ     │    │ تكلفة + قيود + خيارات      │  │
│  │ m130_acados_model.py │    │ m130_ocp_setup.py          │  │
│  └──────────┬──────────┘    └──────────┬─────────────────┘  │
└─────────────┼──────────────────────────┼────────────────────┘
              │                          │
              ▼                          ▼
┌─────────────────────────────────────────────────────────────┐
│           ما يفعله ACADOS تلقائياً (عند البناء)             │
│                                                             │
│  CasADi expressions ──→ توليد كود C ──→ ترجمة gcc ──→ .so  │
│                         c_generated_code/                    │
└──────────────────────────────┬──────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────┐
│           أثناء المحاكاة (كل 10ms)                          │
│                                                             │
│  أدخل الحالة ──→ solver.solve() ──→ اقرأ أمر التحكم       │
│  m130_mpc_autopilot.py                                      │
└─────────────────────────────────────────────────────────────┘
```

## لماذا ACADOS وليس Python عادي؟

| | Python عادي | ACADOS |
|---|---|---|
| السرعة | بطيء (ميلي ثوانٍ - ثوانٍ) | سريع جداً (ميكرو ثوانٍ) |
| المشتقات | يدوية أو عددية | تلقائية ودقيقة (CasADi) |
| القيود | صعبة التطبيق | مدمجة في المحلّل |
| الكود المولّد | لا يوجد | C سريع مخصص |

# 2. النموذج (Model)

## الفكرة الأساسية

النموذج يجيب على سؤال واحد:

> "إذا الصاروخ الآن في حالة معينة (سرعة، زاوية، ارتفاع...) وحرّكت الزعنفة بمقدار معين، كيف ستتغير حالته بعد لحظة؟"

رياضياً:
```
dx/dt = f(x, u, p)

x = الحالة (ماذا يحدث الآن)
u = التحكم (ماذا أفعل)
p = المعاملات (ظروف لا أتحكم بها)
```

الملف المسؤول: `m130_acados_model.py`

---

## 2.1 الحالة x -- "ماذا يحدث الآن"

10 حالات أساسية + 3 إضافية (أوامر سابقة):

```
x = [V, gamma, chi, p, q, r, alpha, beta, phi, h, de_prev, dr_prev, da_prev]
      │    │      │   │  │  │   │      │    │   │    │        │        │
      │    │      │   │  │  │   │      │    │   │    └────────┴────────┘
      │    │      │   │  │  │   │      │    │   │     أوامر سابقة (3)
      │    │      │   └──┴──┘   │      │    │   │
      │    │      │   سرعات     │      │    │   الارتفاع المطبّع (h/100)
      │    │      │   الدوران   │      │    زاوية اللف (roll)
      │    │      │   (rad/s)   │      زاوية الانزلاق الجانبي (sideslip)
      │    │      │             زاوية الهجوم (angle of attack)
      │    │      اتجاه المسار الأفقي (heading)
      │    زاوية مسار الطيران (flight path angle)
      السرعة الكلية (m/s)
```

### لماذا هذه الحالات بالذات؟

لأنها تصف حركة الصاروخ بالكامل في **إطار Wind-Axis**:

| الحالة | الرمز | الوصف | الوحدة |
|---|---|---|---|
| السرعة | V | كم سرعته (سكالر واحد بدل 3 مركبات) | m/s |
| مسار الطيران | gamma | هل يصعد أم ينزل | rad |
| الاتجاه الأفقي | chi | هل يتجه شمالاً أم شرقاً | rad |
| معدل اللف | p | سرعة الدوران حول المحور الطولي | rad/s |
| معدل الميل | q | سرعة الدوران حول محور الميل | rad/s |
| معدل الانعراج | r | سرعة الدوران حول محور الانعراج | rad/s |
| زاوية الهجوم | alpha | كم الأنف مرفوع عن اتجاه الطيران | rad |
| زاوية الانزلاق | beta | كم الأنف منحرف جانبياً | rad |
| زاوية اللف | phi | هل الصاروخ مائل حول محوره | rad |
| الارتفاع | h | الارتفاع فوق الأرض ÷ 100 | -- |

في الكود (CasADi):
```python
V      = ca.SX.sym('V')
gamma  = ca.SX.sym('gamma')
chi    = ca.SX.sym('chi')
p_rate = ca.SX.sym('p_rate')
q_rate = ca.SX.sym('q_rate')
r_rate = ca.SX.sym('r_rate')
alpha  = ca.SX.sym('alpha')
beta   = ca.SX.sym('beta')
phi    = ca.SX.sym('phi')
h      = ca.SX.sym('h')

x = ca.vertcat(V, gamma, chi, p_rate, q_rate, r_rate, alpha, beta, phi, h)
```

> **ملاحظة:** `ca.SX.sym` ينشئ متغيرات **رمزية** -- ليست أرقاماً بعد. CasADi يبني شجرة تعبيرات يستطيع ACADOS لاحقاً حساب المشتقات منها وتوليد كود C.

## 2.2 التحكم u -- "ماذا أفعل"

```
u = [delta_e, delta_r, delta_a]
       │         │         │
       │         │         زعنفة اللف (roll fin)
       │         زعنفة الانعراج (yaw fin)
       زعنفة الميل (pitch fin)
```

هذه هي الأوامر التي يرسلها MPC للزعانف. المحلّل يبحث عن **أفضل قيم** لهذه الثلاثة في كل خطوة زمنية على الأفق.

في الكود:
```python
delta_e = ca.SX.sym('delta_e')
delta_r = ca.SX.sym('delta_r')
delta_a = ca.SX.sym('delta_a')

u = ca.vertcat(delta_e, delta_r, delta_a)
```

## 2.3 المعاملات p -- "ظروف لا أتحكم بها"

```
p = [mass, thrust, rho, Ixx, Iyy, Izz, Mach]
      │      │      │    │    │    │     │
      │      │      │    └────┴────┘     رقم ماخ (نسبة السرعة لسرعة الصوت)
      │      │      │    عزوم العطالة (kg·m²)
      │      │      كثافة الهواء (kg/m³)
      │      قوة الدفع (N)
      الكتلة (kg)
```

هذه تتغير مع الزمن لكنك **لا تتحكم بها**:
- الكتلة تنقص مع احتراق الوقود
- الدفع ينتهي بعد `burn_time`
- الكثافة تنقص مع الارتفاع
- عزوم العطالة تتغير مع نقصان الوقود
- Mach يتغير مع السرعة ودرجة الحرارة

في الكود:
```python
m_param = ca.SX.sym('m')
thrust  = ca.SX.sym('thrust')
rho     = ca.SX.sym('rho')
Ixx     = ca.SX.sym('Ixx')
Iyy     = ca.SX.sym('Iyy')
Izz     = ca.SX.sym('Izz')
Mach    = ca.SX.sym('Mach')

p = ca.vertcat(m_param, thrust, rho, Ixx, Iyy, Izz, Mach)
```

## 2.4 معادلات الحركة f(x, u, p) -- "كيف تتغير الحالة"

هذا هو قلب النموذج. كل حالة لها معادلة تصف كيف تتغير مع الزمن.

### معادلة السرعة: V_dot

```python
V_dot = (thrust * cos(alpha) * cos(beta) - D - m * g * sin(gamma)) / m
```

```
                    thrust × cos(α) × cos(β)
                    ─────────────────────────    ← مركبة الدفع على اتجاه الطيران
                              │
    V_dot =  ─────────────────┼──────────────────
                              │
              - D             │    - m×g×sin(γ)
              ──              │    ─────────────
              السحب           │    مركبة الجاذبية
              (يبطّئ دائماً)  │    (تبطّئ في الصعود، تسرّع في الهبوط)
             ───────────────────────────────────
                              m (الكتلة)
```

**بالعربي:** السرعة تزيد بالدفع وتنقص بالسحب والجاذبية.

- `thrust × cos(α) × cos(β)`: مركبة الدفع في اتجاه الطيران. إذا الأنف مائل (α ≠ 0)، جزء من الدفع يضيع.
- `D`: قوة السحب الايروديناميكي. تتناسب مع V² (السرعة العالية = سحب أكبر).
- `m×g×sin(γ)`: مركبة الجاذبية. في الصعود (γ > 0) تبطّئ، في الهبوط (γ < 0) تسرّع.

### معادلة مسار الطيران: gamma_dot

```python
gamma_dot = ((thrust * (sin(alpha)*cos(phi) + cos(alpha)*sin(beta)*sin(phi))
              + L_pitch * cos(phi)
              - L_yaw * sin(phi))
             / (m * V)
             - g * cos(gamma) / V)
```

```
                  قوة رفع الميل × cos(φ)     ← الرفع الرئيسي يرفع المسار
              +   قوة رفع الانعراج × sin(φ)   ← مساهمة اللف في الرفع
              +   مركبة الدفع العمودية          ← الدفع يساعد لو α ≠ 0
gamma_dot = ──────────────────────────────────
                         m × V
                                               ← "قوة / (كتلة × سرعة) = معدل انحناء المسار"
              - g × cos(γ) / V                 ← الجاذبية تسحب المسار للأسفل دائماً
```

**بالعربي:** المسار ينحني للأعلى بالرفع وينحني للأسفل بالجاذبية. القسمة على `m×V` لأن:
- كتلة أكبر = صعب تغيير المسار
- سرعة أكبر = صعب تغيير المسار (نصف القطر أكبر)

### معادلة الاتجاه الأفقي: chi_dot

```python
chi_dot = ((thrust * (sin(alpha)*sin(phi) - cos(alpha)*sin(beta)*cos(phi))
            + L_pitch * sin(phi)
            + L_yaw * cos(phi))
           / (m * V * cos(gamma)))
```

نفس فكرة gamma_dot لكن في المستوى الأفقي. القوى الجانبية (من beta والزعنفة) تغيّر اتجاه الطيران يميناً أو يساراً. القسمة على `cos(gamma)` لأن في المسار المائل، نصف القطر الأفقي أكبر.

### معادلات الدوران: p_dot, q_dot, r_dot

```python
p_dot = M_roll  / Ixx    # عزم اللف ÷ عطالة اللف
q_dot = M_pitch / Iyy    # عزم الميل ÷ عطالة الميل
r_dot = M_yaw   / Izz    # عزم الانعراج ÷ عطالة الانعراج
```

**بالعربي:** قانون نيوتن الثاني للدوران: **العزم = العطالة × التسارع الزاوي**.

العزوم تأتي من 3 مصادر:

```
M_pitch = q_dyn × S × d × (Cm_alpha  +  Cm_delta  +  Cm_q)
            │      │   │       │            │           │
            │      │   │       │            │           تخميد (يقاوم الدوران)
            │      │   │       │            تأثير الزعنفة ← ما يتحكم به MPC!
            │      │   │       تأثير زاوية الهجوم (استقرار ثابت)
            │      │   القطر المرجعي (0.130 m)
            │      المساحة المرجعية (0.0133 m²)
            الضغط الديناميكي = ½ × ρ × V²
```

في الكود:
```python
# الضغط الديناميكي
q_dyn = 0.5 * rho * V_safe**2

# عزم الميل الكلي
Cm_total_pitch = (Cm_pitch                                      # من alpha
                  + Cmd_pitch                                    # من الزعنفة
                  + Cmq_coeff * (q_rate * d_ref / (2.0 * V_safe)))  # تخميد
M_pitch = q_dyn * S_ref * d_ref * Cm_total_pitch

# عزم الانعراج (نفس البنية -- الصاروخ متماثل محورياً)
Cm_total_yaw = (Cm_yaw + Cmd_yaw + Cnr_coeff * (r_rate * d_ref / (2.0 * V_safe)))
M_yaw = q_dyn * S_ref * d_ref * Cm_total_yaw

# عزم اللف
Cl_total = (Clp_coeff * (p_rate * d_ref / (2.0 * V_safe))  # تخميد
            + Cl_delta_a * delta_a)                          # الزعنفة
M_roll = q_dyn * S_ref * d_ref * Cl_total
```

### معادلتا alpha و beta:

```python
alpha_dot = q_rate - gamma_dot * cos(phi) - chi_dot * cos(gamma) * sin(phi)
beta_dot  = r_rate + gamma_dot * sin(phi) - chi_dot * cos(gamma) * cos(phi)
```

**بالعربي:** زاوية الهجوم alpha هي الفرق بين اتجاه الأنف واتجاه الطيران. تتغير بسبب:
1. **دوران الصاروخ** (`q_rate`) -- الأنف يتحرك
2. **انحناء المسار** (`gamma_dot`) -- المسار يتحرك

```
alpha = اتجاه الأنف - اتجاه المسار
alpha_dot = (حركة الأنف) - (حركة المسار)
```

نفس الفكرة لـ beta في المستوى الأفقي.

### معادلة اللف:

```python
phi_dot = p_rate + (q_rate * sin(phi) + r_rate * cos(phi)) * tan(gamma)
```

هذه معادلة أويلر الحركية. الحد الإضافي `(q*sin(φ) + r*cos(φ))*tan(γ)` يمثل التأثير المتبادل بين الميل والانعراج عندما المسار مائل.

### معادلة الارتفاع:

```python
h_dot = V * sin(gamma) * 0.01
```

**بالعربي:** إذا المسار مائل للأعلى (γ > 0)، الارتفاع يزيد. إذا مائل للأسفل (γ < 0)، الارتفاع ينقص. `× 0.01` لأن h مطبّع (h/100).

### كل المعادلات معاً:

```python
f_expl = ca.vertcat(V_dot, gamma_dot, chi_dot,
                    p_dot, q_dot, r_dot,
                    alpha_dot, beta_dot, phi_dot,
                    h_dot)

model.x = x
model.u = u
model.p = p
model.f_expl_expr = f_expl
```

## 2.5 المعاملات الايروديناميكية -- من أين تأتي الأرقام؟

المعاملات مثل `Cn(alpha, Mach)` و `Cm(alpha, Mach)` ليست ثابتة. هي **متعددات حدود** مُستخلصة من جداول CFD:

```
الشكل العام:
  C(x, M) = a1(M)×x + a3(M)×x³ + a5(M)×x⁵     (للمعاملات الفردية)
  C(x, M) = c0(M) + c2(M)×x² + c4(M)×x⁴         (للمعاملات الزوجية)

حيث:
  ai(M) = k0 + k1×M + k2×M² + k3×M³
```

مثال -- القوة الطبيعية Cn(alpha, Mach):
```python
Cn_a1 = 13.41 + 0.35*M + 5.55*M² + (-1.06)*M³
Cn_a3 = -67.60 + 326.97*M + (-1012.91)*M² + 771.95*M³
Cn_a5 = 352.76 + (-2515.90)*M + 7116.14*M² + (-5328.78)*M³
Cn_pitch = Cn_a1 * α + Cn_a3 * α³ + Cn_a5 * α⁵
```

```
 Cn ↑
    │         /  M=0.85
    │        /
    │       /  /  M=0.6
    │      /  /
    │     /  /  /  M=0.3
    │    /  /  /
    │   /  /  /
    ├──/──/──/──────────→ alpha
    │ /  /  /
    │/  /  /
    │  /  /
```

**لماذا متعددات حدود وليس جداول؟** لأن CasADi يحتاج **تعبيرات رمزية** يستطيع اشتقاقها تلقائياً. الجداول مع interpolation لا يمكن اشتقاقها رمزياً. لذا تُقرّب الجداول بمتعددات حدود من الدرجة 5 (في alpha/delta) والدرجة 3 (في Mach).

### التناظر المحوري

الصاروخ متماثل محورياً، لذا نفس المعاملات تُستخدم لـ pitch و yaw:
```python
# Pitch: Cn(alpha), Cm(alpha), Cmd(delta_e), Cnd(delta_e)
# Yaw:   Cn(beta),  Cm(beta),  Cmd(delta_r), Cnd(delta_r)  ← نفس الدوال!
Cn_yaw  = Cn_a1 * beta_lim + Cn_a3 * bl3 + Cn_a5 * bl5
Cm_yaw  = Cm_a1 * beta_lim + Cm_a3 * bl3 + Cm_a5 * bl5
```

## 2.6 توسيع النموذج: الأوامر السابقة

في `m130_ocp_setup.py`، يُضاف 3 حالات إضافية لتتبع الأوامر السابقة:

```python
de_prev = ca.SX.sym('de_prev')
dr_prev = ca.SX.sym('dr_prev')
da_prev = ca.SX.sym('da_prev')

model.x = ca.vertcat(model.x, de_prev, dr_prev, da_prev)  # 10 + 3 = 13 حالة
model.f_expl_expr = ca.vertcat(
    model.f_expl_expr,
    model.u[0],  # d(de_prev)/dt = delta_e
    model.u[1],  # d(dr_prev)/dt = delta_r
    model.u[2],  # d(da_prev)/dt = delta_a
)
```

**الغرض:** تمكين معاقبة **معدل تغيير** الأوامر في التكلفة (لمنع الذبذبة).

# 3. التكلفة (Cost Function)

## 3.1 ما هي التكلفة؟

التكلفة هي **رقم واحد** يعبّر عن "كم الحل سيء". المحلّل يبحث عن التحكم الذي **يقلل** هذا الرقم.

```
J = مجموع الأخطاء المربعة × الأوزان

  J صغير = "حل جيد: الصاروخ يتبع المسار المطلوب بأقل حركة زعانف"
  J كبير = "حل سيء: الصاروخ بعيد عن المسار أو الزعانف تتأرجح بعنف"
```

---

## 3.2 أنواع التكلفة في ACADOS

### النوع 1: LINEAR_LS (المستخدم حالياً)

```
التكلفة لخطوة واحدة = ||Vx·x + Vu·u - yref||²_W
```

**بالعربي:** في كل خطوة `k` على الأفق:
1. اختر بعض الحالات والتحكمات (عبر المصفوفات `Vx` و `Vu`)
2. اطرح منها المرجع `yref`
3. ربّع الفرق واضربه بالوزن

**`Vx`** -- مصفوفة اختيار الحالات (أي حالات تدخل التكلفة):

```python
ny = 12   # 12 خرج (9 حالات + 3 أوامر)
nx = 13   # 13 حالة

Vx = np.zeros((ny, nx))
Vx[0, 9] = 1.0   # الخرج 0 = h     (الحالة رقم 9)
Vx[1, 1] = 1.0   # الخرج 1 = gamma (الحالة رقم 1)
Vx[2, 2] = 1.0   # الخرج 2 = chi   (الحالة رقم 2)
Vx[3, 3] = 1.0   # الخرج 3 = p     (الحالة رقم 3)
Vx[4, 4] = 1.0   # الخرج 4 = q     (الحالة رقم 4)
Vx[5, 5] = 1.0   # الخرج 5 = r     (الحالة رقم 5)
Vx[6, 6] = 1.0   # الخرج 6 = alpha (الحالة رقم 6)
Vx[7, 7] = 1.0   # الخرج 7 = beta  (الحالة رقم 7)
Vx[8, 8] = 1.0   # الخرج 8 = phi   (الحالة رقم 8)
```

**`Vu`** -- مصفوفة اختيار التحكمات:

```python
Vu = np.zeros((ny, nu))
Vu[9,  0] = 1.0  # الخرج 9  = delta_e
Vu[10, 1] = 1.0  # الخرج 10 = delta_r
Vu[11, 2] = 1.0  # الخرج 11 = delta_a
```

**النتيجة:** المتجه `y` في كل خطوة هو:
```
y = [h, gamma, chi, p, q, r, alpha, beta, phi, delta_e, delta_r, delta_a]
```

### المرجع yref -- "ماذا أريد"

```python
y_ref = [h_ref, gamma_ref, chi_ref, 0, 0, 0, 0, 0, 0, 0, 0, 0]
#         │        │          │      │  │  │  │  │  │  │  │  │
#         ارتفاع  مسار      اتجاه   p  q  r  α  β  φ  δe δr δa
#         مطلوب   مطلوب     مطلوب   =0 =0 =0 =0 =0 =0 =0 =0 =0
```

"أريد: ارتفاع معين، مسار معين، اتجاه معين. وكل شيء آخر (دوران، alpha، beta، زعانف) = صفر."

## 3.3 الأوزان -- "ما الأهم"

```python
W = np.diag([
    100.0,   # h:       "الارتفاع مهم جداً"
    80.0,    # gamma:   "اتباع المسار مهم"
    80.0,    # chi:     "الاتجاه مهم"
    50.0,    # p_rate:  "لا تدور حول المحور بسرعة"
    40.0,    # q_rate:  "لا تتأرجح (ميل)"
    40.0,    # r_rate:  "لا تنعرج بسرعة"
    5.0,     # alpha:   "زاوية هجوم صغيرة مفضّل (لكن ليس ضرورياً)"
    40.0,    # beta:    "لا تنزلق جانبياً -- مهم!"
    60.0,    # phi:     "لا تميل كثيراً"
    3.0,     # delta_e: "لا تستخدم زعنفة pitch كبيرة"
    10.0,    # delta_r: "لا تستخدم زعنفة yaw كبيرة"
    50.0,    # delta_a: "لا تستخدم زعنفة roll كبيرة"
])
```

**التكلفة لخطوة واحدة رقمياً:**

```
J(k) = 100×(h - h_ref)² + 80×(γ - γ_ref)² + 80×(χ - χ_ref)²
     + 50×p² + 40×q² + 40×r²
     + 5×α² + 40×β² + 60×φ²
     + 3×δe² + 10×δr² + 50×δa²
```

**التكلفة الكلية:**
```
J_total = J(0) + J(1) + J(2) + ... + J(79) + J_terminal(80)
          │                                    │
          تكلفة 80 خطوة وسطية                 تكلفة النهاية
```

### ماذا تعني الأوزان عملياً؟

الأوزان هي **أولويات**. وزن أكبر = "هذا أهم، لا تتنازل عنه".

```
مثال: تريد gamma = 0 و alpha = 0
لكن فيزيائياً لا يمكن تحقيق الاثنين معاً بالضبط.

W_gamma = 80, W_alpha = 5:
  → المحلّل يضحّي بـ alpha لصالح gamma
  → النتيجة: gamma ≈ 0.1°, alpha ≈ 3°

لو عكست:
W_gamma = 5, W_alpha = 80:
  → المحلّل يضحّي بـ gamma لصالح alpha
  → النتيجة: gamma ≈ 4°, alpha ≈ 0.2°
```

### لماذا أوزان التحكم مهمة؟

بدون وزن على `delta_e`، المحلّل سيستخدم أقصى زعنفة ممكنة (±20°) لتحقيق أصغر خطأ في gamma. هذا يسبب:
- ذبذبة عنيفة في الزعانف (bang-bang)
- إجهاد ميكانيكي على المشغّلات
- عدم استقرار

الوزن على `delta_e` يقول: "حاول تقلل gamma **بأقل حركة زعنفة ممكنة**".

## 3.5 التكلفة الطرفية (Terminal Cost)

```
J_total = Σ(k=0 to N-1) J_stage(k)  +  J_terminal(N)
          ─────────────────────────     ──────────────
          تكلفة كل خطوة (W, ny=12)     تكلفة النهاية (W_e, ny_e=9)
```

التكلفة الطرفية فيها **9 مخرجات فقط** (بدون التحكمات الثلاث) -- لأنه لا يوجد أمر تحكم في الخطوة الأخيرة:

```python
ny_e = 9
Vx_e = np.zeros((ny_e, nx))
Vx_e[0, 9] = 1.0  # h
Vx_e[1, 1] = 1.0  # gamma
# ... (بدون Vu)
```

**لماذا تكلفة طرفية منفصلة؟**

التكلفة الطرفية تدفع المحلّل للتخطيط **بعيد المدى**:

```
بدون تكلفة طرفية كبيرة:
  المحلّل يقلل الخطأ الآن ← لكن بعد 4 ثوانٍ الوضع سيء
  (يركز على الحاضر ويتجاهل المستقبل)

مع تكلفة طرفية كبيرة:
  المحلّل يقبل خطأ صغير الآن ← لكن بعد 4 ثوانٍ الوضع جيد
  (يخطط للمستقبل)
```

عادة أوزان `W_e` أكبر من `W`:
```python
W_e = np.diag([200, 160, 160, 100, 80, 80, 10, 40, 120])
#              مضاعف أوزان W تقريباً
```

## 3.6 النوع 2: NONLINEAR_LS (غير مستخدم حالياً)

```
التكلفة = ||y(x, u, p) - yref||²_W
```

الفرق: `y` يمكن أن تكون **أي دالة غير خطية** من CasADi، وليس فقط `Vx·x + Vu·u`.

**مثال:** بدل معاقبة alpha مباشرة، عاقب **عامل الحمولة** (load factor):

```python
# load factor = قوة ايروديناميكية ÷ وزن
n_load = q_dyn * S_ref * Cn / (mass * g)

model.cost_y_expr = ca.vertcat(
    h,
    gamma - gamma_ref,
    n_load,           # ← دالة غير خطية! لا يمكن التعبير عنها بـ Vx·x
    beta,
    delta_e,
)
```

**لماذا هذا أفضل؟**
- `alpha = 5°` في سرعة منخفضة = حمولة صغيرة (آمن)
- `alpha = 5°` في سرعة عالية = حمولة كبيرة (خطير!)
- `LINEAR_LS` لا يفرّق بينهما، `NONLINEAR_LS` يفرّق

## 3.7 النوع 3: EXTERNAL (غير مستخدم حالياً)

```
التكلفة = أي تعبير CasADi تريده
```

حرية كاملة. مثلاً: تكلفة الوقت (time-optimal)، أو تكلفة الوقود، أو تكلفة مخصصة بالكامل. لكنه أبطأ في الحساب لأنه يحتاج Hessian حقيقي.

# 4. القيود (Constraints)

## 4.1 لماذا القيود؟

التكلفة تقول **"أفضّل هذا"**. القيود تقول **"يجب هذا"**. الفرق كبير:

```
التكلفة: "أفضّل alpha صغيرة"     → المحلّل قد يقبل alpha = 25° إذا "يستحق"
القيد:   "alpha ≤ 20° دائماً"    → المحلّل لن يتجاوز 20° أبداً
```

---

## 4.2 قيود التحكم (صلبة -- Hard Constraints)

```python
delta_max = np.radians(20.0)
ocp.constraints.lbu   = np.array([-delta_max, -delta_max, -delta_max])  # حد أدنى
ocp.constraints.ubu   = np.array([+delta_max, +delta_max, +delta_max])  # حد أقصى
ocp.constraints.idxbu = np.array([0, 1, 2])  # أي تحكمات تتأثر: delta_e, delta_r, delta_a
```

**بالعربي:** "الزعنفة لا يمكنها فيزيائياً أن تتحرك أكثر من ±20°." هذا قيد **صلب** -- لا يمكن انتهاكه أبداً لأنه يمثل حداً فيزيائياً حقيقياً (المشغّل لا يدور أكثر من هذا).

```
delta_e
  +20° ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ← سقف صلب (لا يمكن التجاوز)
        \  /\     /\
         \/  \   /  \ /\
              \ /    x  \
               x    / \  \
  -20° ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ← أرضية صلبة
```

---

## 4.3 قيود الحالة (مرنة -- Soft Constraints)

```python
p_max     = np.radians(60.0)
q_max     = np.radians(80.0)
r_max     = np.radians(60.0)
alpha_max = np.radians(15.0)
beta_max  = np.radians(10.0)
phi_max   = np.radians(15.0)

ocp.constraints.lbx   = np.array([-p_max, -q_max, -r_max, -alpha_max, -beta_max, -phi_max])
ocp.constraints.ubx   = np.array([+p_max, +q_max, +r_max, +alpha_max, +beta_max, +phi_max])
ocp.constraints.idxbx = np.array([3, 4, 5, 6, 7, 8])  # p, q, r, alpha, beta, phi
```

### ما الفرق بين صلب ومرن؟

```
قيد صلب (Hard):
  "alpha ≤ 15°"
  ← إذا لم يجد المحلّل حلاً يحقق هذا، يفشل (status ≠ 0, INFEASIBLE)

قيد مرن (Soft):
  "alpha ≤ 15° ... لكن إذا اضطررت تجاوزها، ادفع غرامة"
  ← المحلّل يمكنه تجاوز 15° لكن يدفع "ثمناً" في التكلفة
```

**لماذا نستخدم قيوداً مرنة؟** لأن الصاروخ قد يمر بلحظات يستحيل فيها الحفاظ على `alpha ≤ 15°` (مثل عصفة هواء). القيد الصلب يجعل المحلّل يفشل. القيد المرن يقول "تجاوزها مؤقتاً وارجع بسرعة".

```
alpha
  15° ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ← الحد المرن
                  /\
                 /  \  ← انتهاك مؤقت (يدفع غرامة)
        /\      /    \
  ─────/──\────/──────\──────
      /    \  /        \
```

### تفعيل المرونة (Slack Variables)

```python
ocp.constraints.idxsbx = np.array([0, 1, 2, 3, 4, 5])  # كل القيود الست مرنة
```

### العقوبات

```python
ocp.cost.Zl = np.array([5000.0, 1000.0, 1000.0, 2000.0, 5000.0, 8000.0])  # عقوبة تربيعية
ocp.cost.Zu = np.array([5000.0, 1000.0, 1000.0, 2000.0, 5000.0, 8000.0])
ocp.cost.zl = np.array([500.0,  100.0,  100.0,  200.0,  500.0,  800.0])   # عقوبة خطية
ocp.cost.zu = np.array([500.0,  100.0,  100.0,  200.0,  500.0,  800.0])
```

**كيف تعمل العقوبات؟**

```
J_slack = zl × s + Zl × s²

حيث s = مقدار الانتهاك (صفر إذا لم يُنتهك القيد)
```

- `zl` (خطية): غرامة ثابتة لكل وحدة انتهاك.
  "كل درجة فوق الحد = 500 وحدة عقوبة"
- `Zl` (تربيعية): غرامة تزداد مع مربع الانتهاك.
  "درجة واحدة فوق الحد = 5000 × 1² = 5000"
  "درجتان فوق الحد = 5000 × 4 = 20000" ← أربعة أضعاف!

هذا يمنع الانتهاكات الكبيرة بشدة:

```
انتهاك   عقوبة خطية   عقوبة تربيعية   المجموع
  0°         0              0              0
  1°        500           5000           5500
  2°       1000          20000          21000    ← 4× أكثر من 1°
  5°       2500         125000         127500    ← لن يحدث عملياً
```

### ترتيب العقوبات حسب الحالة

| الفهرس | الحالة | Zl | zl | المعنى |
|---|---|---|---|---|
| 0 | p (لف) | 5000 | 500 | عقوبة عالية -- الدوران خطير |
| 1 | q (ميل) | 1000 | 100 | عقوبة متوسطة |
| 2 | r (انعراج) | 1000 | 100 | عقوبة متوسطة |
| 3 | alpha | 2000 | 200 | عقوبة متوسطة-عالية |
| 4 | beta | 5000 | 500 | عقوبة عالية -- الانزلاق خطير |
| 5 | phi (لف) | 8000 | 800 | أعلى عقوبة -- الميل الكبير خطير جداً |

## 4.4 قيد الحالة الابتدائية

```python
solver.set(0, "lbx", x_mpc)
solver.set(0, "ubx", x_mpc)
```

**بالعربي:** "الحالة في الخطوة 0 = القياس الحالي **بالضبط**."

هذا هو **الرابط بين المحلّل والواقع**. بدونه، المحلّل سيتخيل حالة وهمية.

```
          x_mpc (القياس الحالي من المستشعرات)
            │
            ▼
  خطوة 0 ──→ خطوة 1 ──→ خطوة 2 ──→ ... ──→ خطوة 80
  (مثبتة)    (حرة)       (حرة)               (حرة)
  "أنت هنا"  "ماذا سيحدث لو..."              "...بعد 4 ثوانٍ"
```

`lbx = ubx = x_mpc` يعني: الحد الأدنى = الحد الأقصى = القياس. أي القيمة مثبتة (مساواة).

# 5. خيارات المحلّل (Solver Options)

## 5.1 خوارزمية NLP: SQP مقابل SQP_RTI

### ما هو SQP (Sequential Quadratic Programming)؟

خوارزمية لحل المسألة غير الخطية عن طريق تحويلها لسلسلة مسائل تربيعية (QP).

**تشبيه:** أنت على جبل في الضباب وتريد الوصول للقاع:
1. تحسس الأرض حولك (linearize)
2. افترض أن الأرض مستوية مائلة مع انحناء (تقريب QP)
3. احسب أفضل خطوة (حل QP)
4. اخطُ
5. كرر حتى تصل القاع (تقارب)

```
SQP الكامل:
  while (residual > tolerance):
      QP ← linearize NLP حول الحل الحالي
      حل QP → خطوة تحديث
      حدّث الحل
      احسب residual
  return الحل الأمثل
```

**في الكود:**
```python
ocp.solver_options.nlp_solver_type = 'SQP'
ocp.solver_options.nlp_solver_max_iter = 5  # حد أقصى 5 تكرارات
```

### ما هو SQP_RTI (Real-Time Iteration)؟

نفس SQP لكن **تكرار واحد فقط** في كل خطوة تحكم.

**الفلسفة:** بدلاً من حل المسألة بالكامل والانتظار، أرسل **أفضل ما عندك الآن** وحسّن في الخطوة التالية.

```
SQP الكامل (ينتظر حتى يتقارب):
  t = 0.00s:  alpha=5° → [تكرار1 → تكرار2 → تكرار3] → delta_e = 3.1°
  t = 0.03s:  ← أول أمر بعد 30ms تأخير!

RTI (تكرار واحد فقط):
  t = 0.00s:  alpha=5° → [تكرار1] → delta_e = 2.5°  ← ليس أمثل لكن سريع
  t = 0.01s:  alpha=3.8° → [تكرار1] → delta_e = 2.9°  ← تحسّن
  t = 0.02s:  alpha=1.5° → [تكرار1] → delta_e = 3.0°  ← تحسّن أكثر
  t = 0.03s:  alpha=0.4° → [تكرار1] → delta_e = 3.1°  ← وصل!
```

**لماذا يعمل RTI؟** بين خطوة والتي بعدها (10ms)، الصاروخ لا يتغير كثيراً. فالحل السابق تخمين ممتاز للحل الجديد. RTI "يلاحق" الحل الأمثل -- متأخر بخطوة واحدة دائماً لكن الفرق صغير.

### المقارنة

| | SQP كامل | SQP_RTI |
|---|---|---|
| عدد التكرارات | حتى يتقارب (3-20) | **واحد فقط** |
| جودة الحل | أمثل | تقريبي (يتحسن مع الوقت) |
| زمن التنفيذ | متغير | ثابت ومتوقع |
| الاستخدام | offline أو dt كبير | real-time، كل ميلي ثانية مهمة |
| الفلسفة | "أوجد الأفضل ثم نفّذ" | "تحسّن قليلاً كل خطوة" |

**تشبيه:** SQP = تقف عند كل منعطف وتحسب الزاوية المثالية ثم تدير المقود. RTI = تدير المقود باستمرار بأفضل تقدير لديك في كل لحظة.

---

## 5.2 محلّلات NLP المتاحة

| الخوارزمية | الوصف | متى تُستخدم |
|---|---|---|
| `SQP` | تكرارات كاملة حتى التقارب | تريد حل دقيق، الوقت متاح |
| `SQP_RTI` | تكرار واحد فقط | تحكم real-time |
| `DDP` | Differential Dynamic Programming | مسائل بدون قيود path |

## 5.3 محلّل QP (المستوى الأدنى)

بعد التخطيط (linearization)، كل تكرار SQP يحل مسألة QP. هذه المسألة تربيعية التكلفة وخطية القيود -- لها حل مباشر وسريع.

```python
ocp.solver_options.qp_solver = 'PARTIAL_CONDENSING_HPIPM'
ocp.solver_options.qp_solver_iter_max = 200
```

### ما هو Condensing؟

المسألة الأصلية كبيرة:
```
80 خطوة × 13 حالة = 1040 متغير حالة
80 خطوة × 3 تحكم  = 240  متغير تحكم
المجموع = 1280 متغير
```

**Condensing** يزيل متغيرات الحالة (لأنها مرتبطة بالتحكمات عبر معادلات الحركة):

| الطريقة | عدد المتغيرات | المصفوفات | الأفضل عندما |
|---|---|---|---|
| No Condensing | 1280 | شريطية (sparse) | -- |
| Full Condensing | 240 | كثيفة (dense) | N < 20 |
| **Partial Condensing** | ~400 | مختلطة | **N > 20 (حالتنا)** |

### ما هو HPIPM؟

**H**igh-**P**erformance **I**nterior **P**oint **M**ethod -- خوارزمية متخصصة في حل QP الناتجة من MPC. تستغل البنية الشريطية (banded structure) للمسألة. تعمل فوق **BLASFEO** (مكتبة جبر خطي محسّنة على مستوى التعليمات).

### محلّلات QP المتاحة

| المحلّل | الوصف |
|---|---|
| `PARTIAL_CONDENSING_HPIPM` | **الأسرع عادة** (المستخدم حالياً) |
| `FULL_CONDENSING_HPIPM` | أفضل لأفق قصير N < 20 |
| `FULL_CONDENSING_QPOASES` | بديل أبطأ لكن أكثر متانة (robustness) |
| `FULL_CONDENSING_DAQP` | بديل خفيف |

## 5.4 التكامل (Integrator)

```python
ocp.solver_options.integrator_type = 'ERK'
ocp.solver_options.sim_method_num_stages = 4  # RK4
```

المحلّل يحتاج أن يتنبأ: "إذا أعطيت delta_e = 5°، أين سيكون الصاروخ بعد 50ms؟"

للإجابة، يجب **تكامل** المعادلات التفاضلية `dx/dt = f(x,u,p)`.

### ERK (Explicit Runge-Kutta)

بدل خطوة أويلر البسيطة (`x_new = x + f(x)×dt`)، يعمل 4 تقييمات وسطية:

```
k1 = f(x)                           ← ميل في البداية
k2 = f(x + dt/2 × k1)               ← ميل في المنتصف (باستخدام k1)
k3 = f(x + dt/2 × k2)               ← ميل في المنتصف (باستخدام k2)
k4 = f(x + dt × k3)                 ← ميل في النهاية

x_new = x + dt/6 × (k1 + 2×k2 + 2×k3 + k4)
```

`num_stages = 4` = RK4 الكلاسيكي. دقة من الرتبة الرابعة (الخطأ ∝ dt⁴).

### طرق التكامل المتاحة

| الطريقة | الوصف | متى تُستخدم |
|---|---|---|
| `ERK` | Runge-Kutta صريح | **سريع** -- يحتاج نموذج explicit (حالتنا) |
| `IRK` | Runge-Kutta ضمني | للأنظمة الصلبة (stiff systems) |
| `GNSF` | بنية خاصة | أسرع لبعض الأنظمة ذات البنية المحددة |

## 5.5 تقريب Hessian

```python
ocp.solver_options.hessian_approx = 'GAUSS_NEWTON'
```

SQP يحتاج مصفوفة المشتقات الثانية (Hessian) لتشكيل QP. حسابها الحقيقي مكلف.

| الطريقة | الوصف | السرعة | الدقة |
|---|---|---|---|
| `GAUSS_NEWTON` | تقريب (يعمل فقط مع Least-Squares) | **سريع** | كافي لـ MPC |
| `EXACT` | Hessian حقيقي | أبطأ | أدق -- للمسائل الصعبة |

`GAUSS_NEWTON` يعمل بتقريب: `H ≈ Jᵀ × W × J` حيث `J` هو Jacobian دالة التكلفة. هذا صالح فقط مع تكلفة من نوع Least-Squares (وهو حالتنا).

```python
ocp.solver_options.N_horizon = 80  # عدد الخطوات
```

```
الآن                                                بعد 4 ثوانٍ

 [0]──[1]──[2]──[3]── ... ──[78]──[79]──[80]
  │    │    │    │            │     │     │
```

## 5.7 التسامحات (Tolerances)

```python
ocp.solver_options.nlp_solver_tol_stat = 1e-2   # شرط الأمثلية (stationarity)
ocp.solver_options.nlp_solver_tol_eq   = 1e-2   # شرط المساواة (dynamics feasibility)
ocp.solver_options.nlp_solver_tol_ineq = 1e-2   # شرط القيود (constraint satisfaction)
ocp.solver_options.nlp_solver_tol_comp = 1e-2   # شرط التكامل (complementarity)
```

هذه تقول: "متى يتوقف SQP ويعتبر أنه تقارب؟"

| التسامح | المعنى |
|---|---|
| `tol_stat` | "هل نحن عند نقطة حيث لا يمكن تحسين التكلفة أكثر؟" |
| `tol_eq` | "هل معادلات الحركة محققة بدقة كافية؟" |
| `tol_ineq` | "هل القيود محققة بدقة كافية؟" |
| `tol_comp` | "هل شروط التكاملية للقيود المرنة محققة؟" |

```
tol = 1e-2 (حالياً):   الخطأ < 1%       ← فضفاض
tol = 1e-4:             الخطأ < 0.01%    ← معقول
tol = 1e-6:             الخطأ < 0.0001%  ← دقيق جداً
```

تسامح فضفاض = سرعة أكبر لكن حل أقل دقة.
تسامح ضيق = حل أدق لكن تكرارات أكثر.

# 6. Warm-Start و Shifting

## 6.1 المشكلة

كل 10ms تحل مسألة بـ 240 متغير تحكم. من أين تبدأ البحث؟

## 6.2 Cold Start مقابل Warm Start

```
Cold Start (بداية باردة):
  ابدأ من تخمين عشوائي أو صفري
  → بطيء -- يحتاج تكرارات كثيرة للتقارب

Warm Start (بداية دافئة):
  ابدأ من الحل السابق
  → سريع -- الحل الجديد قريب جداً من القديم
```

**لماذا Warm Start يعمل؟** لأنك تحل مسألة شبه متطابقة كل 10ms. الحالة تغيرت قليلاً، المرجع تغير قليلاً. فالحل الأمثل الجديد قريب جداً من السابق.

## 6.3 التهيئة الأولى (في الكود)

```python
if not self._warm_started:
    # ضع الحالة الحالية في كل خطوات الأفق
    for k in range(N + 1):
        solver.set(k, "x", x_mpc)
    # ضع تحكم صفري
    u_init = np.zeros(NU)
    for k in range(N):
        solver.set(k, "u", u_init)
    # حل 3 مرات لتحسين التخمين الأولي
    for _ in range(3):
        solver.solve()
    self._warm_started = True
```

## 6.4 Shifting (الإزاحة)

بعد كل حل ناجح، تعمل "إزاحة" للأفق كتخمين أولي للمرة القادمة:

```
الحل عند t = 1.00s:
  [u(0), u(1), u(2), ..., u(79)]
   │     │     │           │
   t=1.00 t=1.05 t=1.10 ... t=4.95

التخمين عند t = 1.01s (بعد الإزاحة):
  [u(1), u(2), u(3), ..., u(79), u(79)]  ← إزاحة يسار + تكرار الأخير
   │     │     │           │      │
   t=1.05 t=1.10 t=1.15 ... t=4.95 t=5.00
```

في الكود:
```python
# بعد حل ناجح: انقل الحل خطوة للأمام
for k in range(N - 1):
    x_next = solver.get(k + 1, "x")
    u_next = solver.get(k + 1, "u")
    if np.all(np.isfinite(x_next)):
        solver.set(k, "x", x_next)
    if np.all(np.isfinite(u_next)):
        solver.set(k, "u", u_next)

# الخطوة الأخيرة: كرر آخر حالة
x_last = solver.get(N, "x")
if np.all(np.isfinite(x_last)):
    solver.set(N - 1, "x", x_last)
```

## 6.5 إعادة التهيئة عند الفشل المتكرر

إذا فشل المحلّل مرات كثيرة متتالية، يُعاد تهيئته بالكامل:

```python
def _reinitialize_solver(self, x_mpc):
    N = self.N_horizon
    solver = self._solver
    for k in range(N + 1):
        solver.set(k, "x", x_mpc)       # أعد ضبط كل الحالات
    for k in range(N):
        solver.set(k, "u", np.zeros(NU)) # صفّر كل التحكمات
    self._consecutive_fails = 0
```

هذا "cold start" إجباري -- آخر ملاذ عندما warm start لم يعد يعمل.

## 7.4 ماذا يحدث داخل solver.solve()

```
1. التكامل (ERK / RK4):
   ← يتنبأ بالحالة لكل 80 خطوة على الأفق
   ← باستخدام معادلات الحركة المولّدة ككود C

2. Linearization:
   ← يحسب Jacobians (المشتقات) عند كل خطوة
   ← "لو غيّرت delta_e قليلاً، كم يتغير alpha؟"

3. تشكيل QP:
   ← يحوّل المسألة غير الخطية إلى مسألة تربيعية
   ← 80 × 3 = 240 متغير تحكم

4. Partial Condensing:
   ← يضغط المسألة لتقليل حجمها
   ← من ~1280 متغير إلى ~400 متغير

5. HPIPM:
   ← يحل QP المضغوطة
   ← باستخدام Interior-Point Method
   ← يستغل البنية الشريطية للسرعة

6. تحديث الحل:
   ← يُحدّث الحالات والتحكمات على كامل الأفق

   → SQP: كرر الخطوات 1-6 حتى التقارب
   → RTI: توقف بعد مرة واحدة → أخرج u_opt
```
